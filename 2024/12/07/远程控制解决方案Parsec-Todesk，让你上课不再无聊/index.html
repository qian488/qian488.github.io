远程控制解决方案Parsec + Todesk，让你上课不再无聊

远程控制解决方案Parsec + Todesk，让你上课不再无聊

2024-12-07

parsec, todesk

前言

首先，这两种解决方案都不需要固定在同一网络下，所以才会是我首先选择的，但是为什么会有两个呢？其实就是各自都有些问题，需要互补。综合体验下来，我认为两者配合使用，是不错的选择。而且都是免费，虽然todesk付费可以解决挺多问题的，但是，秉着能白嫖绝不花钱的心思，这里我们仅需要下载免费的个人版。

Todesk

简单直接，开箱直用。去官网ToDesk远程桌面软件下载，然后注册账号登陆，设置连接密码，两个设备都登录同一个账号，就可以在两个设备上直接连接了。

记得在设置中选择可以使用安全密码进行连接

todesk的优势之处就在于:

  • 不限于设备，手机、平板、其他系统的电脑，均可使用。

  • 连接相对比较稳定

但是免费版的缺点就是

  • 画质较差

  • 不能接手柄等外设操控（得加钱）

以我个人的使用经验来看，使用平板远控敲代码、看书、看视频都是不错的选择，上课的时候还可以挂着游戏刷刷刷

注，todesk还有远程开机功能，但是需要有另外一个设备登录账号并且与被控设备在同一网络下，且需要去电脑bios开启一个功能，我忘了，总之原理就是类似于路由器通过网络唤醒电脑，需要确保这个功能有开启，才能实现远程开机。

我个人闲比较鸡肋，想了一个算麻烦也不算麻烦的解决方案，选择去bios设置通电定时开机，并且写一个小工具检测电脑网络，没有网络重置网络适配器，一般情况下，都是可以确保开机就有网络，这个工具设置在电脑开机的时候就自动启动就行了。

Parsec

parsec相对来说会比较麻烦，但是功能强大很多。前往官网Download Parsec | Parsec下载，注册账号，登录。并且，需要自己准备一个魔法/科学上网工具。主要用于打开软件登录，远控的时候可以不需要。

如果出现-800或者-6023等报错，可以前往以下路径C:\Users\Awith\AppData\Roaming\Parsec去修改配置文件config.txt或者config.json，在txt里面更改后保存后会自动到json里，

app_proxy_address = 127.0.0.1
app_proxy_scheme = http
app_proxy = true
app_proxy_port = 7890

这里的7890是你使用的魔法/科学上网工具的端口。

这里的log文件记录了程序的日志，如果有其他报错，可以查看日志，再搜搜是什么问题。目前，我只遇到过-800和-6023，但是还是有些时候会连不上，什么都没改变的情况下，有时又可以连上，我还在看看是什么情况。

如果找不到这个文件路径以及没有扩展名，记得在查看选项框内，勾选文件扩展名和隐藏的项目。

parsec的操作也很简单，两边启动parsec，点击connect就行。

parsec基本上各方面完爆todesk，前提是网络必须比较好，在网络都差的情况下，parsec的画质甚至不如todesk。

在网络条件好的情况下，画质在远控中表现得很不错，而且也比较稳定，并且还可以连接手柄、触控板等外接输入设备。实现远程打游戏完全不是问题。唯一的问题就是，只有pc端，ipad等其他设备并没有适配。

写在最后

我使用parsec的使用场景基本上就是远控开发游戏和打游戏，其他场景下基本使用todesk即可满足。至少，以目前我的使用场景来看，parsec+todesk已经是非常不错的远程控制解决方案。parsec除了不能在其他设备上进行远控和连接有时会被墙，刚需好网络，其他的所有体验都是完爆todesk，至于todesk付费的功能，能做到什么程度我不知道。

另外，如果需要被控电脑不被查看显示的内容，可以买个显卡欺瞒器，插上去，然后关闭物理显示器，就可以实现隐私屏的功能了。
